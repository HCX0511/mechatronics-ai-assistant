# SPI 通信实验

## 一、实验定位

SPI 是一种高速同步串行通信方式，常用于显示屏、Flash、ADC、DAC 等外设。

## 二、实验目标

- 理解 SPI 的主从结构
- 掌握 MOSI、MISO、SCK、CS 的作用
- 掌握时钟极性和时钟相位配置
- 学会完成主从收发数据

## 三、实验器材

- 单片机开发板
- SPI 外设模块
- 下载器与连接线

## 四、实验原理

SPI 通过时钟同步收发数据，常见信号包括：

- `MOSI`
- `MISO`
- `SCK`
- `CS/NSS`

SPI 常为全双工通信，主机发送的同时也可能接收数据。

## 五、接线与硬件细节

- 主机和从机要共地
- 片选信号必须明确控制
- 不同外设对 CPOL/CPHA 可能要求不同

## 六、程序设计思路

1. 初始化 SPI 引脚和外设
2. 设置主机模式
3. 配置时钟分频、位宽、CPOL、CPHA
4. 拉低片选
5. 发送命令并读取数据
6. 拉高片选结束通信

## 七、平台差异

### 1. 51 单片机

- 常见实验中可能没有专用 SPI 外设
- 可能采用软件模拟时序

### 2. STM32

- 硬件 SPI 配置更完整
- 更常见于工程项目

### 3. Arduino

- 使用 `SPI` 库快速初始化和收发

## 八、常见问题与排查

### 1. 没有响应

- 片选无效
- 时钟没输出
- 模式不匹配

### 2. 数据错位

- CPOL/CPHA 设置错误
- 位宽设置不对

### 3. 读值异常

- 指令格式不符合器件手册
- 读写顺序不对

## 九、排查建议

1. 先查片选
2. 再查时钟模式
3. 再查发送与接收顺序

## 十、安全注意事项

- 不要带电频繁插拔模块
- 多从机共享总线时，片选不能冲突

## 十一、报告写作建议

重点可写：

- SPI 四线含义
- 时钟模式解释
- 主从收发实验现象
- 数据错位原因分析

## 十二、拓展方向

- SPI Flash 读写
- TFT 屏驱动
- 与 ADC/DAC 模块联调
