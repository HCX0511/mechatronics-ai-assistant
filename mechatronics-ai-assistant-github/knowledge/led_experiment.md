# LED 流水灯实验

## 一、实验定位

LED 流水灯是嵌入式入门实验中最基础、也最重要的实验之一。  
它的价值不只是“把灯点亮”，而是帮助学生建立以下认识：

- 软件可以直接控制硬件输出
- GPIO 是最基础的外设接口
- 延时、循环、位操作会直接影响外设现象
- 一个完整实验包含接线、配置、编程、下载、调试、观察、分析全流程

## 二、实验目标

- 理解 GPIO 输出控制的基本方法
- 理解 LED 的导通条件和点亮逻辑
- 掌握顺序点亮、延时控制和循环结构
- 能独立完成 LED 流水效果程序
- 学会分析“全亮、全灭、不流水、顺序错乱”等典型问题

## 三、实验器材

- 单片机开发板
- 板载 LED 或 8 个外接 LED
- 限流电阻
- 下载器
- 面包板与连接线
- 电源模块
- 编程软件与下载工具

## 四、实验原理

### 1. LED 点亮原理

LED 是发光二极管，只有在两端形成合适电压并有电流流过时才会点亮。  
因此程序控制 LED 的本质，是控制对应 GPIO 引脚输出高电平或低电平，改变 LED 回路是否导通。

### 2. 高电平点亮与低电平点亮

开发板上 LED 常见有两种连接方式：

- 高电平点亮：GPIO 输出高电平时 LED 亮
- 低电平点亮：GPIO 输出低电平时 LED 亮

这一点一定要先确认，否则程序逻辑可能完全反着写。

### 3. 流水效果形成原因

所谓“流水灯”，其实就是多个 LED 按照设定顺序依次点亮和熄灭。  
当相邻 LED 的点亮时间差合适时，人眼会看到一个连续“移动”的亮点。

## 五、接线与硬件细节

### 1. 接线检查重点

- 每个 LED 是否都串联限流电阻
- LED 正负极是否接对
- GPIO 与 LED 编号是否一一对应
- 电源和地线是否连接正确
- 下载器和开发板是否共地

### 2. 典型错误

- 多个 LED 共用一个错误控制线
- LED 极性接反
- 使用了错误电压
- 程序里的引脚号与实际接线不一致

## 六、程序设计思路

### 1. 最小功能验证

建议先不要一上来就写完整流水灯，而是先做这三步：

1. 点亮一个 LED
2. 让一个 LED 闪烁
3. 再扩展到多个 LED 顺序点亮

### 2. 标准流程

1. 初始化 GPIO 为输出模式
2. 明确点亮电平
3. 定义点亮顺序
4. 每次只改变一个 LED 状态
5. 在状态切换之间加入延时
6. 用循环反复执行

### 3. 伪代码示例

```text
初始化 GPIO
while(1):
    点亮 LED1
    延时
    熄灭 LED1
    点亮 LED2
    延时
    熄灭 LED2
    ...
```

## 七、不同平台实现差异

### 1. 51 单片机

- 常直接操作端口寄存器
- 更适合理解位操作和底层控制
- 延时函数通常自己写或直接空循环

### 2. STM32

- 先配时钟，再配 GPIO
- 常见写法是 HAL 库或寄存器方式
- 更接近现代嵌入式开发流程

### 3. Arduino

- 使用 `pinMode()`、`digitalWrite()` 等函数
- 上手快，适合演示
- 底层细节被封装较多

## 八、实验现象

正常现象：

- LED 按顺序依次点亮
- 点亮时间和间隔稳定
- 修改延时后，流水速度能明显变化

异常现象：

- 全亮
- 全灭
- 顺序错乱
- 闪烁不稳定
- 速度过快或过慢

## 九、常见问题与排查

### 1. LED 全都不亮

可能原因：

- 没有供电
- GPIO 未初始化
- LED 极性接反
- 程序未下载成功

排查建议：

1. 先查供电
2. 再查单个 LED 能否点亮
3. 再查程序是否真正运行

### 2. LED 全部同时亮

可能原因：

- 点亮逻辑写反
- 多个 LED 连接方式错误
- 程序一次性把所有引脚都置为有效电平

### 3. 不流水，只是闪一下

可能原因：

- 延时太短
- 主循环逻辑有问题
- 程序执行后被复位或卡死

### 4. 流水方向反了

可能原因：

- 接线顺序与代码顺序不一致
- 引脚编号对应错误

## 十、安全注意事项

- 接线前先断电
- LED 必须串联限流电阻
- 不要让电源正极和地线直接短接
- 开发板异常发热时立即断电
