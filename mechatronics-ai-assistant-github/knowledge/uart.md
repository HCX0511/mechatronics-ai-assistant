# UART 串口通信实验

## 一、实验定位

UART 是嵌入式系统中最常用的通信接口之一。  
它不仅用于模块通信，也广泛用于调试输出，因此是必须掌握的基础实验。

## 二、实验目标

- 理解异步串行通信原理
- 掌握波特率、数据位、停止位、校验位概念
- 学会初始化串口并收发数据
- 学会使用串口助手进行联调

## 三、实验器材

- 单片机开发板
- USB 转串口模块或板载串口
- 串口助手软件
- 下载器与连接线

## 四、实验原理

UART 常见一帧数据包括：

- 起始位
- 数据位
- 可选校验位
- 停止位

UART 是异步通信，所以通信双方必须约定好波特率和帧格式，否则就会出现乱码或收发失败。

## 五、接线与硬件细节

- 单片机 TX 接对方 RX
- 单片机 RX 接对方 TX
- 双方必须共地
- 注意电平兼容问题

如果开发板是 3.3V 逻辑，不能随意接不兼容的 5V 串口模块。

## 六、程序设计思路

1. 配置串口时钟和引脚复用
2. 设置波特率
3. 设置数据位、停止位、校验位
4. 编写发送函数
5. 编写接收逻辑
6. 用串口助手做回显或数据交互

### 伪代码示例

```text
初始化 UART
while(1):
    如果收到数据:
        回传收到的数据
```

## 七、平台差异

### 1. 51 单片机

- 常通过 SCON、TMOD、TH1 等配置串口
- 常借助定时器产生波特率

### 2. STM32

- 常配置 USART 波特率寄存器
- 常结合中断、DMA 使用

### 3. Arduino

- 常用 `Serial.begin()`、`Serial.print()` 等接口
- 非常适合快速验证收发逻辑

## 八、常见问题与排查

### 1. 串口乱码

可能原因：

- 波特率不一致
- 时钟配置错误
- 数据位、停止位不一致

### 2. 收不到数据

可能原因：

- TX/RX 接反
- 串口号选错
- 地线没共地
- 程序没有进入接收逻辑

### 3. 只能发不能收

可能原因：

- 只初始化了发送
- 接收中断或缓冲逻辑不完整

### 4. 回显不稳定

可能原因：

- 串口接收缓存处理不及时
- 发送过快
- 中断优先级或时序处理不当

## 九、排查建议

1. 先统一串口助手参数
2. 再检查 TX/RX 和共地
3. 再检查程序是否执行到发送和接收逻辑
4. 最后再查中断、缓存和波特率误差

## 十、安全注意事项

- 串口模块电平要与开发板匹配
- 不要错误把供电口当信号口连接

## 十一、报告写作建议

重点可写：

- UART 帧格式
- 波特率设置方法
- 收发实验现象
- 乱码和收不到数据的原因分析

### 可直接参考的现象描述

“程序下载后，单片机能够按设定波特率向串口助手发送数据，并可正确接收上位机输入内容。参数一致时通信稳定，参数不一致时会出现乱码现象。”
